Assignment 3: Securing Your Website

Due: 9:00am Monday 1st Wednesday 3rd October 2018

In previous assignments, you have designed and implemented a website using the technologies introduced in the last eight weeks. In this assignment, you are asked to secure your website by adding some security features.

Requirements

General requirements include:

  • converting certain characters that have special meaning in an HTML context to HTML Entities to minimize the possibility of XSS attacks where appropriate.
  • using md5/SHA and salt (you choose your salt) to encrypt users login passwords before inserting in Oracle database
  • using md5/SHA and the chosen salt to validate password when user logs in, which means users password is hashed and compared with the database copy. If they match, the user is authenticated.
  • using captcha to implement contact form on your website.
  • listing the file names and the changes you made to enhance the secuirty of your site in a word/pdf file or attaching the info in the SIT203 Assignment Submission Form.

Assessment will be made based on

  • the reliability of filtering special characters in users input;
  • the reliability of using MD5/SHA hashing algorithm to encode data and validate data;
  • the reliability of using captcha to collect information generated by person.
